# Протокол эксперимента

## Гипотеза

Обучение на ошибках текущего чекпоинта Uni-MuMER должно лучше переноситься на
неизвестные изображения формул, чем обучение на корпусе ошибок, который не
изменяется после первоначального построения.

Измеряется качество прямого распознавания рукописных математических выражений.
Поиск и исправление ошибок используются только как задачи supervised
fine-tuning. Метод не относится к обучению с подкреплением.

## Разделение данных

Нормализованный MathWriting один раз делится на две части:

| Часть | Доля | Назначение |
| --- | ---: | --- |
| Train pool | 95% | Обучение распознаванию, поиск ошибок и построение корпусов |
| Validation | 5% | Метрики, выбор чекпоинта и ранняя остановка |

Для разделения используется `seed=42`. Файл `assignments.jsonl` хранит
назначение каждого примера, а `summary.json` содержит количество примеров и
контрольные отпечатки. Изображения validation не могут попасть в записи с
ошибками, KEEP, обычное распознавание или replay.

Внешними тестами служат CROHME 2014, 2016 и 2019 из официального архива
Uni-MuMER. Они используются только для финального отчёта и не влияют на выбор
чекпоинта.

## Сравниваемые варианты

### B0: frozen

Опубликованный чекпоинт `phxember/Uni-MuMER-Qwen2.5-VL-3B` без адаптации. Его
предсказания используются как парная база для подсчёта исправленных и
испорченных примеров.

### B1: recognition_lora

Одна эпоха обычного LoRA-обучения на задаче преобразования изображений
MathWriting в LaTeX.

### B2: static_error

Корпус `D0` один раз строится по предсказаниям B0 на всём train pool. Каждый
следующий цикл LoRA использует тот же неизменный `D0`. Количество циклов B2
равно номеру чекпоинта, выбранного для B3.

### B3: dynamic_error

Первый чекпоинт обучается на том же `D0`, что и B2. После каждого цикла текущий
чекпоинт заново проходит весь train pool и создаёт новый корпус `Dk`. Выбирается
чекпоинт с максимальным `net_fixed_count` на validation.

Таким образом, B2 и B3 начинают с одинаковой базовой модели, конфигурации LoRA
и начального корпуса. Основное различие состоит в обновлении корпуса ошибок.

## Динамический цикл

Пусть `Gk` обозначает текущую модель, `T` обозначает неизменный train pool, а
`Dk` обозначает корпус, построенный по предсказаниям `Gk`.

1. `Gk` выполняет распознавание каждого изображения из `T`.
2. Предсказания сравниваются с эталонным LaTeX.
3. Ошибки характеризуются CER, типом редактирования, уверенностью и приоритетом части train pool.
4. `Dk` строится из актуальных ошибок, KEEP, обычного распознавания и replay.
5. Одна эпоха LoRA-обучения на `Dk` создаёт `Gk+1`.
6. `Gk+1` оценивается на неизменной validation.

У корпуса нет фиксированного ограничения по количеству примеров. Число строк и
фактическое количество шагов оптимизатора записываются в `summary.json` и
`train_metrics.json`. Одно исходное изображение даёт не более одной записи за
цикл. Изображение из предыдущего цикла пропускает один цикл.

Train pool имеет пять детерминированных частей `S1...S5`. Они поочерёдно
получают повышенный приоритет при ранжировании, но поиск ошибок всегда
выполняется на полном train pool.

## Обучающие задачи

Точные инструкции Uni-MuMER определены в `src/dec_unimumer/prompts.py`:

- `error_find`: разметка ошибочных фрагментов токенами `<error_start>`,
  `<error_end>` и `<deleted>`;
- `error_fix`: формирование операций замены, вставки или удаления и вывод
  исправленной формулы;
- `keep`: подтверждение уже правильного предсказания через задачу поиска ошибки;
- `recognition`: обычное преобразование изображения в LaTeX;
- `replay`: повторное использование ошибки из более раннего цикла.

В режиме `alternate` актуальные ошибки детерминированно распределяются между
`error_find` и `error_fix`. На validation и CROHME используется только промпт
прямого распознавания.

## Конфигурация

```text
Базовая модель:                 phxember/Uni-MuMER-Qwen2.5-VL-3B
Точность вычислений:            bfloat16
Языковые модули LoRA:           q/k/v/o + gate/up/down projections
Визуальные модули LoRA:         qkv
Ранг / alpha LoRA:              8 / 16
Dropout LoRA:                   0.05
Эпох на цикл корпуса ошибок:    1
Эпох обычного распознавания:    1
Размер micro-batch:             1
Накопление градиента:           8
Максимальная длина:             без обрезания
Ограничение изображения:        160000 пикселей
Seed:                           42
```

Параметры базового чекпоинта заморожены. Обучаются LoRA-адаптеры в языковой и
визуальной частях модели.

## Метрики

Для эталона `y*`, предсказания B0 `y0` и текущего предсказания `y`:

- **ExpRate**: точное совпадение после нормализации;
- **CDM**: официальный Character Detection Matching из UniMERNet;
- **CER**: нормализованное посимвольное расстояние редактирования;
- **BLEU**: корпусный BLEU по токенам LaTeX;
- **Исправлено**: `y0 != y*` и `y == y*`;
- **Испорчено**: `y0 == y*` и `y != y*`;
- **Чистый прирост**: `Исправлено - Испорчено`.

Все метрики вычисляются на MathWriting validation во время обучения. Финальные
отчёты формируются отдельно для каждого года CROHME. Сводная таблица использует
арифметическое среднее ExpRate, CDM, CER и BLEU по трём наборам, а количества
исправленных и испорченных примеров суммируются.

## Сохраняемые артефакты

Полный запуск сохраняет:

- назначения частей данных, seed и контрольные отпечатки;
- предсказания B0 на train pool, validation и трёх тестах;
- манифест выбранных примеров и состав каждого `Dk`;
- `train_metrics.json` и validation-предсказания каждого чекпоинта;
- финальные предсказания и парные отчёты B0-B3 для каждого CROHME;
- `reports/mathwriting/comparison.json` и `comparison.csv`;
- `outputs/mathwriting/experiment_summary.json`.
